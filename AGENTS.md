# Agent 工作约定

## Git 版本控制规则

1. **修改前**：如果工作区有未提交的更改，先检查 `git status` 确认状态。
2. **修改后**：完成一组相关修改（一个完整功能/修复）后，**必须执行 `git add -A && git commit -m "<描述>"`**。
3. **提交信息规范**：
   - 新增功能：`feat: <描述>`
   - 修复问题：`fix: <描述>`
   - 重构代码：`refactor: <描述>`
   - 文档/配置：`docs: <描述>` 或 `chore: <描述>`
4. **禁止**：不要提交 `.pyc`、临时文件或大型二进制结果文件（已存在的除外）。

## 项目背景

大学生创新训练计划项目：融合信息熵和深度学习的复杂化工过程风险预测方法研究。
