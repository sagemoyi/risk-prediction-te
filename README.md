## 一、项目概况

### 1.1 我们在做什么？

想象一个化工厂，里面有几十个传感器（温度、压力、流量等）在实时监测。如果这些传感器的数据出现异常，就可能意味着要出事故了。

**我们的目标**是：用计算机分析这些传感器数据，提前预测出风险有多大。

具体来说，我们要做三件事：
1. **把复杂数据变简单**：把几十个传感器的数据，通过数学方法压缩成一条"风险曲线"
2. **用人工智能预测**：用神经网络学习这条风险曲线的规律，预测未来的风险值
3. **比较不同方法**：比较三种神经网络（LSTM、Bi-LSTM、Attention-Bi-LSTM）哪个预测得更准

### 1.2 技术流程（一张图看懂）

```
原始传感器数据（22个变量）
    ↓
【步骤1】小波去噪 —— 去掉数据里的"噪声"
    ↓
【步骤2】滑动窗口 —— 把数据切成一段一段的
    ↓
【步骤3】相关分析 —— 看哪些传感器之间有关联
    ↓
【步骤4】复杂网络 —— 把关联画成一张"关系网"
    ↓
【步骤5】结构熵 —— 计算这张网的"混乱程度"
    ↓
【步骤6】归一化 —— 把混乱程度转换成0~1之间的"风险值"
    ↓
【步骤7】监督样本 —— 做成神经网络能学习的格式
    ↓
【步骤8】神经网络训练 —— 让AI学会预测风险
    ↓
【步骤9】对比评估 —— 看哪个AI预测得最准
```

## 五、关于真实数据（重要！）

### 5.1 现在的数据是真的吗？

**不是**。目前代码里用的是**模拟数据**（计算机随机生成的），用来验证整个流程能不能跑通。

模拟数据和真实数据的区别：
- 模拟数据：程序自动生成的，能看到流程是否正常运行
- 真实数据：来自 TEP（田纳西-伊斯曼过程）仿真平台，是论文里实际使用的数据

### 5.2 真实数据长什么样？

真实 TEP 数据一般是这样：
- **格式**：`.mat`（MATLAB格式）、`.dat`（文本格式）、或 `.csv`（表格格式）
- **变量**：52个或53个变量，其中前41个是测量变量（XMEAS1~XMEAS41），后11个是操纵变量（XMV1~XMV11）
- **采样**：每3分钟采一次样
- **长度**：训练集500个点（25小时），测试集960个点（48小时）
- **工况**：
  - 正常工况（No Fault）
  - 20种故障工况（Fault 1 ~ Fault 20）

论文里只用了 **XMEAS1~XMEAS22** 这22个变量。

### 5.3 怎么拿到真实数据？

**方法一（最推荐）：找指导老师要**

绝大多数做化工过程控制的课题组都有 TEP 数据。直接问老师："老师，咱们课题组有没有 TEP 的仿真数据？我们需要 No Fault、Fault 4 和 Fault 11 的。"

**方法二：网上下载**

1. **MathWorks 官网**（MATLAB官方）：
   - 访问 https://www.mathworks.com/help/predmaint/ug/chemical-process-fault-detection-deep-learning.html
   - 里面有4个 `.mat` 文件可以下载
   - 但需要 MATLAB 才能打开

2. **华盛顿大学官网**（原始数据源）：
   - 访问 https://depts.washington.edu/control/LARRY/TE/download.html
   - 可以下载 Fortran 仿真代码，自己跑数据

3. **GitHub 上的开源数据集**：
   - 搜索 "Tennessee Eastman Process dataset"
   - 注意：很多仓库用了 Git LFS（大文件存储），下载下来的可能是"指针文件"，不是真实数据

**方法三：用 MATLAB 自己生成**

如果你们有 MATLAB：
1. 去华盛顿大学网站下载 TEP 仿真代码
2. 在 MATLAB 里运行仿真
3. 保存数据为 `.mat` 或导出为 `.csv`

### 5.4 拿到真实数据后怎么用？

**方法：修改 `src/risk_pipeline.py` 里的两个地方**

**第1步**：把 `DATA_SOURCE` 从 `'simulated'` 改成 `'csv'`：

```python
# 在 src/risk_pipeline.py 第55行附近
DATA_SOURCE = 'csv'  # 改成这个
```

**第2步**：修改 `CSV_PATHS` 指向你的真实数据文件：

```python
CSV_PATHS = {
    'normal_path':  'data/normal.csv',
    'fault4_path':  'data/fault4.csv',
    'fault11_path': 'data/fault11.csv',
}
```

**对数据文件的要求**：
- CSV 文件包含至少 `XMEAS1` ~ `XMEAS22` 这22列
- 列名可以是 `XMEAS1, XMEAS2, ...` 或 `xmeas_1, xmeas_2, ...`
- 每个文件是一维时间序列（一行一个时间点，一列一个变量）

修改完这两处后，直接运行 `python src/risk_pipeline.py` 即可。

如果数据是 `.mat` 格式（MATLAB文件），建议先用 MATLAB 导出为 CSV：
```matlab
% 在 MATLAB 中
writetable(your_table, 'data.csv')
```

其他代码完全不用改。

## 六、每一步的原理（大白话版）

### 步骤1：小波去噪（Wavelet Denoising）

**问题**：传感器数据里总有"噪声"——就像你用手机录歌，背景里总有杂音。

**做法**：小波去噪就像给数据"降噪耳机"。它把数据拆成不同频率的成分，把高频的噪声部分压低，再把有用的信号拼回来。

**参数**：
- `db4`：用的一种叫"Daubechies 4"的小波基函数（论文里只说"小波函数"，没说具体用哪种，所以我们用了最常用的一种）
- `level=3`：拆成3层

### 步骤2：滑动窗口（Sliding Window）

**问题**：我们不能一次性看所有数据，因为数据太长了。而且风险是随时间变化的。

**做法**：就像看股票K线图，我们不看每一天的细节，而是看"最近150天"的整体趋势。我们让窗口从左滑到右，每次滑1步。

**参数**：
- `window_size=150`：每个窗口看150个时间点
- `step_size=1`：每次只往前滑1步

### 步骤3：相关分析（Spearman Correlation）

**问题**：22个传感器之间，谁和谁有关系？比如反应器温度高了，冷却水流量是不是也会变？

**做法**：计算每两个传感器之间的"相关程度"。Spearman 是一种排名相关，它不在乎具体数值，只在乎变化趋势是否一致。

**结果**：得到一个 22×22 的相关矩阵。

### 步骤4：复杂网络（Complex Network）

**问题**：相关矩阵数字太多，看不懂。

**做法**：如果两个传感器的相关程度超过 0.2，就在它们之间画一条线。这样就把数字变成了一张"关系网"（图）。

**参数**：
- `threshold=0.2`：相关程度≥0.2才连边

### 步骤5：结构熵（Structural Entropy）

**问题**：怎么量化这张网的"混乱程度"？

**做法**：看每个传感器连了多少条线（这叫"度"）。如果所有传感器的连线数量差不多，网就比较"均匀"，熵值高；如果有的传感器连线很多、有的很少，网就比较"不均匀"，熵值低。

**公式**：`E = -Σ (k_i/Σk_i) · ln(k_i/Σk_i)`
- `k_i`：第 i 个传感器的连线数
- `Σk_i`：所有传感器的连线总数

### 步骤6：归一化（Normalization）

**问题**：不同工况算出来的熵值范围不一样，没法直接比较。

**做法**：把所有工况的熵值放在一起，找到最小值和最大值，把所有熵值按比例缩放到 0~1 之间。

**关键**：论文明确说是**跨工况统一归一化**——把 No Fault、Fault 4、Fault 11 的熵全部混在一起求最大最小值。这样不同工况的风险才能放在同一个尺度上比较。

### 步骤7：风险分级（Risk Grading）

**做法**：把 0~1 的风险值分成4档：
- 0~0.3：低风险
- 0.3~0.6：较低风险
- 0.6~0.9：较高风险
- 0.9~1.0：高风险

### 步骤8：监督样本（Supervised Samples）

**问题**：神经网络不知道怎么"直接看"一条风险曲线，它需要输入输出对。

**做法**：比如用过去12个时间点的风险值作为"输入"，下一个时间点的风险值作为"输出"。就像老师给你12道例题，让你预测第13题的答案。

**参数**：
- `lookback=12`：看过去12步
- 训练集6000条，测试集3000条

## 七、常见问题（FAQ）

### Q1：运行时报错 "ModuleNotFoundError: No module named 'xxx'"

**解决**：这个模块没装。在命令行里运行 `pip install xxx` 即可。比如：
```bash
pip install pandas
```

### Q2：图里中文显示为方块/乱码

**解决**：代码已经内置了中文字体（`src/SourceHanSansSC-Regular.otf`）。如果还显示方块，说明你的系统不支持这个字体。可以：
1. 在 `src/risk_pipeline.py` 开头找到字体设置部分
2. 把 `font_path = 'src/SourceHanSansSC-Regular.otf'` 改成你系统里有的中文字体路径

Windows 常见中文字体路径：`C:/Windows/Fonts/simhei.ttf`（黑体）

### Q3：运行很慢怎么办？

**原因**：滑动窗口步长为1，数据量很大（12000个点产生约11850个窗口）。

**解决**：如果只是测试代码能不能跑通，可以改小参数：
- 把 `n_samples = 12000` 改成 `n_samples = 3000`
- 或者把 `step_size = 1` 改成 `step_size = 5`（每5步取一个窗口）

注意：中期答辩展示时，还是用 `step_size = 1` 的结果更好看。

### Q4：怎么把图放到 PPT 里？

1. 打开 `figures/` 文件夹
2. 右键 `02_risk_sequences.png` → 复制
3. 打开 PPT，在空白页右键 → 粘贴
4. 拖动图片边缘调整大小

### Q5：老师问"这是不是真实数据？"

**诚实回答**："目前我们先用模拟数据把整个 pipeline 跑通了，验证流程的正确性。真实数据正在向老师/网上获取中，拿到后只需替换数据读取部分，其他代码完全不用改。"

## 八、名词解释

| 名词 | 简单解释 |
|------|---------|
| **TEP** | 田纳西-伊斯曼过程（Tennessee Eastman Process），一个虚拟的化工厂仿真模型，学术界用它来测试各种算法 |
| **XMEAS** | "测量变量"（Measured Variable），就是传感器实际测到的数据，比如温度、压力 |
| **小波去噪** | 一种信号处理方法，像"降噪耳机"一样去掉数据里的杂音 |
| **滑动窗口** | 把长数据切成一段一段来分析，就像看股票时看"最近30天"而不是"全部历史" |
| **Spearman 相关** | 衡量两个变量变化趋势是否一致的一种方法 |
| **邻接矩阵** | 一张表格，记录网络里谁和谁有连线。0表示没连线，1表示有连线 |
| **结构熵** | 衡量一张网络"混乱/均匀程度"的数字。越均匀熵越高，越不均匀熵越低 |
| **归一化** | 把不同范围的数据缩放到同一个范围（比如0~1），方便比较 |
| **监督学习** | 给AI看"输入+正确答案"的例子，让它学会从输入预测输出 |
| **LSTM** | 一种能记住长期信息的神经网络，适合处理时间序列 |
| **Bi-LSTM** | 两个 LSTM 背靠背，一个从左往右看，一个从右往左看 |
| **Attention** | 让AI学会"把注意力放在重要的时刻上" |
| **lookback** | 看过去多少个时间点 |
| **epoch** | 把整个训练数据完整看一遍 |
| **batch** | 每次给AI看多少条数据 |
