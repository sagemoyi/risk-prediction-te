**标题**：化工过程安全风险的实时预测需求

**内容要点**（3 个 bullet，每行不超过 15 字）：

1. **化工过程高风险性**：反应条件苛刻，多变量耦合强，故障后果严重

**底部一句话**（本页核心信息）：
> 本项目面向 Tennessee Eastman Process（TEP）仿真数据，探索一种"特征提取 → 深度学习预测"的完整风险预测流程。

**设计建议**：

### 第 6 页：工作方法（1）风险序列生成

**标题**：基于复杂网络与结构熵的风险序列提取

**内容**（左右分栏）：

**左栏（网络构建）**：
- 滑动窗口：time window = 150，moving step = 1
- 相关分析：Spearman 秩相关系数
- 邻接矩阵：阈值 0.2 → 无向二值矩阵

**右栏（熵与归一化）**：
- 结构熵定义（公式1）：
  ```
  E = -Σ (k_i/Σk_i) · ln(k_i/Σk_i)
  ```
- 归一化（公式5）：跨工况统一 min-max → [0,1]
- 风险分级：低 / 较低 / 较高 / 高（4 档）

**设计建议**：
- 公式不要太大，能看清即可
- 右侧可以放一张"复杂网络示意图"（22个节点带连边）
- "跨工况统一"这四个字加粗或标橙（这是你们复核后的关键认知）

---

### 第 7 页：工作方法（2）预测模型

**标题**：Attention-based Bi-LSTM 实时风险预测

**内容**（三部分）：

**1. Bi-LSTM**：
- 2 层双向 LSTM，隐层节点 128
- 同时捕获前向与后向时序依赖

**2. 加性注意力机制（Additive Attention）**：
- 公式：`score = a · <s_{i-1}, h_j>`
- 对全部时间步隐藏状态加权求和，聚焦关键时段

**3. 训练设置**：
- 训练/测试：6000 / 3000（Fault 11 单工况）
- Batch size = 32，Epoch = 500
- 损失函数：MSE；激活函数：tanh

**设计建议**：
- 放一张模型结构示意图：输入 → Bi-LSTM → Attention → FC → 输出
- 注意力权重可用热力图示意（即使还没跑出来，放一个示意性小图也很加分）

### 背景页过渡

> 化工过程具有高度非线性和多变量耦合的特点，一旦发生故障后果严重。传统的统计预测方法难以处理复杂耦合关系，而单一的深度学习模型又缺乏可解释性。因此，我们考虑将结构熵的可解释性与 Attention-Bi-LSTM 的时序建模能力相结合，探索一条新的风险预测路径。

### 技术路线页过渡

> 这是我们项目的总体技术路线。左侧是风险序列生成模块，右侧是深度学习预测模块。核心思路是：先把 22 维过程数据通过复杂网络和结构熵压缩成一维风险序列，再用 Attention-Bi-LSTM 做实时预测。

### 成果页过渡

> 目前我们已经完成了前三项工作，风险序列生成模块已经基本跑通，LSTM 和 Bi-LSTM 基线模型也已搭建完成。Attention-Bi-LSTM 正在调试中，预计五月底完成。

## 容易被问到的问题（提前准备）

| 问题 | 建议回答要点 |
|------|-------------|
| "你们和原文的区别是什么？" | 我们是复现核心思路 + 简化实现，重点验证方法在 TEP 上的可行性，不是追求创新点。 |
| "结构熵为什么能反映风险？" | 结构熵衡量网络异质性。正常运行时网络结构相对稳定，故障时变量关联关系改变，导致熵值变化，从而刻画风险状态。 |
| "为什么只选 3 个工况？" | 先跑通最小闭环，Fault 11 用于训练预测，No Fault 和 Fault 4 用于风险分布对比。后续时间允许再扩展。 |
| "Attention 的作用是什么？" | 对长时序风险序列，Attention 能自适应地聚焦对当前预测最重要的历史时刻，提升预测精度。 |
| "如果模型效果不如预期怎么办？" | 先检查数据划分和归一化是否正确；再调参；如果仍有限，在论文中如实分析原因，这也是一种有价值的结论。 |
| "你们能按时结题吗？" | 能。核心模块已完成大半，后续按甘特图推进，时间充裕。 |
