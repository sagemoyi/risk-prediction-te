# 蓝本论文可实现清单

对应论文：*Real-time risk prediction of chemical processes based on Attention-based Bi-LSTM*  
整理时间：2026-04-21  
用途：把论文思路拆成可实现模块，供项目推进、分工和写代码时直接使用。

## 1. 先说结论

这篇论文的核心不是“直接拿原始 TEP 多变量数据做深度学习预测”，而是先把多变量过程数据压缩成一条一维风险序列，再对这条风险序列做时序预测。

主线可以压缩成一句话：

`TEP 多变量数据 -> 去噪/预处理 -> 滑动时间窗口 -> 相关网络 -> 网络结构熵 -> 相对风险序列 -> Attention-Bi-LSTM -> 风险预测结果`

如果你们能把这条主线真实跑通，就已经完成了论文的核心可实现部分。

## 2. 这份清单怎么读

我把内容拆成三层：

- `必须做`：不做就不算在复现论文主线。
- `建议先做简化版`：先跑通项目最小闭环，再决定要不要贴近论文。
- `必须回原文确认`：摘要里没给够细节，写代码前必须去 PDF 正文核对。

说明：

- 下文中凡是标注“推断”的内容，表示这是根据论文摘要和相关前序工作推出来的，不等于论文正文逐字写法。
- 当前不建议把论文之外的扩展方法硬塞进主线。

## 3. 论文主线拆解

### 模块 A：数据准备

#### 目标

拿到可用于风险建模的 `TEP` 多变量时序数据。

#### 输入

- 正常工况数据
- 若干故障工况数据

#### 输出

- 按时间顺序排列的多变量矩阵
- 每个工况对应的标签或文件编号

#### 必须做

- 确认使用的是哪一版 `TEP` 数据；
- 明确每个样本的采样间隔、变量数量、正常/故障工况编号；
- 把正常工况和故障工况分开管理。

#### 建议先做简化版

- 第一版只选 `3` 到 `5` 个典型故障；
- 先不追求覆盖全部 `21` 类故障。

#### 必须回原文确认

- 论文使用了多少个过程变量；
- 论文是否使用全部变量，还是筛选后使用；
- 论文具体选了哪些故障工况做展示。

## 4. 模块 B：数据预处理与去噪

#### 目标

降低原始过程数据噪声，得到更稳定的后续网络构建输入。

#### 输入

- 原始多变量时间序列

#### 输出

- 去噪后的多变量时间序列

#### 必须做

- 缺失值、异常值、时间顺序检查；
- 标准化策略确定；
- 去噪方法固定下来。

#### 论文明确提到

- 论文摘要明确提到对温度、压力、流量、液位等多源过程数据进行去噪预处理。

#### 推断

- 相关前序工作很可能使用了 `wavelet denoising`，也就是小波去噪。

#### 建议先做简化版

- 第一版先试 `标准化 + 简单平滑` 或 `小波去噪` 二选一；
- 只要风险序列能稳定生成，就先往下走。

#### 必须回原文确认

- 去噪方法到底是不是小波去噪；
- 使用了哪种小波基函数；
- 阈值函数、分解层数、重构方式是什么。

## 5. 模块 C：滑动窗口切片

#### 目标

把连续时序切成多个局部时间窗口，为“变量间相关网络”建模做准备。

#### 输入

- 去噪后的多变量时间序列

#### 输出

- 一组按时间推进的窗口数据块

#### 必须做

- 定义窗口长度 `window_size`；
- 定义移动步长 `step_size`；
- 保证每个窗口都能产生一张网络。

#### 论文明确提到

- 摘要明确写了设置 `time windows` 和 `moving step lengths`。

#### 建议先做简化版

- 第一版先固定一个窗口长度和步长，不做参数搜索；
- 跑通流程后再做敏感性分析。

#### 必须回原文确认

- 窗口长度是多少；
- 移动步长是多少；
- 是否对不同工况使用相同参数。

## 6. 模块 D：变量相关性分析

#### 目标

在每个时间窗口内，计算变量之间的相关关系。

#### 输入

- 单个窗口内的多变量数据

#### 输出

- 相关系数矩阵

#### 必须做

- 明确采用哪种相关系数；
- 明确相关矩阵如何转为邻接矩阵。

#### 推断

- 你们申报书里写的是 `Spearman` 秩相关；
- 论文作者同年的前序论文也使用了 `Spearman's rank correlation coefficient`；
- 因此当前最合理的实现选择就是先用 `Spearman`。

#### 建议先做简化版

- 先不做太复杂的多关系融合；
- 先生成一张无向加权网络或者阈值化后的无向网络。

#### 必须回原文确认

- 当前论文到底是只用 `Spearman`，还是还用了其他关联规则；
- 邻接矩阵是保留权重，还是二值化；
- 相关阈值怎么取。

#### 当前务实建议

- 第一版实现：窗口内计算 `Spearman` 相关矩阵，再按绝对值阈值生成邻接矩阵。

## 7. 模块 E：复杂网络构建

#### 目标

把窗口内变量关系表示成一张随时间演化的复杂网络。

#### 输入

- 变量相关矩阵或邻接矩阵

#### 输出

- 每个窗口对应一张图网络

#### 必须做

- 节点定义：每个过程变量一个节点；
- 边定义：两个变量存在显著相关关系时建立连边；
- 图结构保存：用于后续熵值计算。

#### 建议先做简化版

- 第一版优先使用 `无向图`；
- 第一版先不做风险溯源和根因识别，只服务于生成风险序列。

#### 必须回原文确认

- 图是无向图还是有向图；
- 是否保留边权；
- 是否做网络稀疏化。

## 8. 模块 F：网络结构熵计算

#### 目标

把每个窗口的复杂网络压缩成一个标量，表示当下网络结构复杂度或风险状态。

#### 输入

- 每个时间窗口的复杂网络

#### 输出

- 每个窗口一个熵值

#### 必须做

- 明确熵值公式；
- 对每个窗口独立计算；
- 得到一条按时间排列的熵值序列。

#### 论文明确提到

- 摘要明确写了 `network structure entropy`；
- 关键词也包含 `Network structure entropy`。

#### 必须回原文确认

- 结构熵的精确定义；
- 公式里使用的是节点度、介数，还是其他网络指标；
- 是否引用 `Zhang et al., 2016` 的某个具体公式；
- 是否对熵值做额外平滑或变换。

#### 当前务实建议

- 这一步不要凭印象乱写公式；
- 真正开始编码前，必须把论文正文中的熵值公式抄出来，单独存成一页。

## 9. 模块 G：相对风险序列生成

#### 目标

把结构熵序列归一化成一维相对风险序列，作为深度学习模型输入。

#### 输入

- 按窗口排列的结构熵序列

#### 输出

- 一维相对风险序列

#### 论文明确提到

- 摘要明确写了通过 `max-min deviation standardization` 得到 `1D relative risk sequence`。

#### 必须做

- 实现极差标准化；
- 确保风险序列与时间窗口一一对应；
- 检查正常与故障工况下风险序列是否有可区分性。

#### 建议先做简化版

- 第一版先画图，不急着训练模型；
- 先看正常和故障工况下风险曲线有没有明显差别。

#### 必须回原文确认

- 归一化公式具体写法；
- 风险序列数值范围是否固定在 `[0,1]`；
- 是否按单个工况分别归一化，还是全局归一化。

## 10. 模块 H：预测任务定义

#### 目标

把相对风险序列转换成监督学习样本。

#### 输入

- 一维相对风险序列

#### 输出

- 模型训练样本 `(X, y)`

#### 必须做

- 明确输入序列长度 `lookback`；
- 明确输出是“下一时刻风险值”还是“未来多步风险值”；
- 定义训练集、验证集、测试集切分。

#### 当前务实建议

- 第一版任务就做单步回归预测：
  - 输入：过去 `lookback` 步风险值
  - 输出：下一步风险值

#### 必须回原文确认

- 论文到底做的是单步预测还是多步预测；
- 训练集、测试集如何划分；
- 是否按工况分开训练还是合并训练。

## 11. 模块 I：基线模型

#### 目标

建立可比较的时序预测基线。

#### 建议顺序

1. `LSTM`
2. `Bi-LSTM`
3. `Attention-Bi-LSTM`

#### 必须做

- 三种模型共享同一份数据处理结果；
- 统一输入长度、损失函数、训练轮数和评价指标；
- 保证对比公平。

#### 建议先做简化版

- 先让 `LSTM` 和 `Bi-LSTM` 跑通；
- `Attention-Bi-LSTM` 放在最后接入。

#### 必须回原文确认

- LSTM 隐层维度；
- 层数；
- dropout 是否使用；
- batch size、epoch、optimizer、learning rate；
- 注意力层放在哪一层输出之后。

## 12. 模块 J：Attention-Bi-LSTM

#### 目标

在 `Bi-LSTM` 输出上加入注意力机制，提高对关键时间步的建模能力。

#### 输入

- 风险序列样本窗口

#### 输出

- 下一时刻或未来若干步风险预测值

#### 必须做

- 先实现纯 `Bi-LSTM`；
- 再在其输出序列上加 attention；
- 保证 attention 权重可视化或至少可导出。

#### 当前务实建议

- 第一版实现可采用常见时序 attention 结构：
  - `Bi-LSTM hidden states -> attention weights -> context vector -> fully connected -> prediction`

#### 必须回原文确认

- 论文里的 attention 是哪种形式；
- 是单头、加性还是点积式；
- 是对全部时间步做加权还是仅对最终隐状态做处理。

## 13. 模块 K：评价指标与结果展示

#### 目标

验证目标模型是否优于基线，并能支持结题材料。

#### 论文摘要明确暗示

- 提升了化工过程风险预测精度；
- 比现有技术更好。

#### 当前务实建议

主指标先统一为：

- `RMSE`
- `MAE`
- `R²`

图表至少包括：

- 正常工况与故障工况风险序列图；
- 真实值与预测值对比图；
- 三种模型指标对比表；
- 如果能做出来，再补 attention 权重可视化。

#### 必须回原文确认

- 论文实际使用了哪些指标；
- 是否还有 `MAPE`、`MSE` 或其他指标；
- 对比对象除了 `LSTM`、`Bi-LSTM` 外还有哪些。

## 14. 模块 L：结题需要保留的中间产物

这些东西后面写结题论文时非常关键，建议从一开始就保存：

## 15. 建议你们按这个顺序实现

### 第一阶段：只做风险序列

目标：

- 不碰深度学习；
- 只做 `数据 -> 网络 -> 熵值 -> 风险序列`；
- 确认正常/故障工况风险曲线确实有区别。

阶段完成标志：

- 至少有两张图能说明风险序列不是乱的。

### 第二阶段：先做最弱基线

目标：

- 用风险序列训练一个最简单的 `LSTM`；
- 看是否能做单步预测。

阶段完成标志：

- 至少能输出预测曲线和一个指标表。

### 第三阶段：再加 Bi-LSTM

目标：

- 把 `Bi-LSTM` 接上；
- 比较是否优于 `LSTM`。

### 第四阶段：最后加 Attention

目标：

- 接入 `Attention-Bi-LSTM`；
- 在统一数据和统一评估框架下做对比。

## 16. 当前最容易卡住的地方

### 卡点 1：结构熵公式不清楚

处理原则：

- 不要凭参考文献标题猜公式；
- 单独回 PDF 核对；
- 核对完再编码。

### 卡点 2：风险序列做出来没有区分度

处理原则：

- 先检查窗口长度；
- 再检查相关阈值；
- 再检查归一化方式；
- 最后再怀疑模型。

### 卡点 3：一开始就想做所有故障

处理原则：

- 先少量工况跑通；
- 跑通后再扩展。

### 卡点 4：模型做了很多，数据定义却没定

处理原则：

- 先定输入输出、预测步长、数据切分；
- 再写模型代码。

## 17. 当前建议的最小实现版本

如果只为了尽快建立项目闭环，我建议先做这个版本：

1. 选 `TEP` 正常工况 + `3` 个故障工况；
2. 做基础标准化 + 一种固定去噪；
3. 固定一个滑动窗口和步长；
4. 用 `Spearman` 相关生成邻接矩阵；
5. 按论文正文公式计算结构熵；
6. 用极差标准化得到风险序列；
7. 做单步风险回归预测；
8. 比较 `LSTM`、`Bi-LSTM`、`Attention-Bi-LSTM`。

这个版本已经足够支撑：

- 项目主线；
- 结题论文；
- 答辩展示；
- 后续扩展。
