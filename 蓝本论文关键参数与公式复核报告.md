## 逐项复核详情与原文对应

### 1. 小波基函数与阈值细节

**原文定位：**
- 摘要第 2 句："multisource process data ... are preprocessed for denoising"
- 第 3.1 节："Denoising preprocessing **based on wavelet function** is done to multisource process data"
- 第 4.2.1 节："Denoising preprocessing is done to the 22 process variables ... **with the use of the wavelet function**"

**复核情况：**
PDF 中 **仅出现 "wavelet function" 这一笼统表述**，未涉及：
- 小波基函数名称（如 db4、sym8 等）
- 阈值函数（如 soft threshold、hard threshold）
- 分解层数（如 3 层、4 层）
- 重构方式

**结论：** 仍属待确认项，首版若需实现，只能采用工程默认值（如 db4 + soft threshold + 3 层），**不可在论文中声称“完全按原文参数复现”**。

### 2. 结构熵定义方式

**原文定位：**
- 第 2.1 节："Eq. (1) shows that network structure entropy (E) is defined in accordance with the node degree of the complex network model"
- 公式(1)：
  ```
  E = - Σ_{i=1}^{N} (k_i / Σ_{i=1}^{N} k_i) * ln(k_i / Σ_{i=1}^{N} k_i)
  ```
- 说明："where, k_i refers to the degree of the network node i, and N is the number of network nodes."

**复核情况：**
公式排版、符号、定义与核对表预估值完全一致。结构熵就是**按节点度分布计算的 Shannon 信息熵**。

**结论：** 已确认，可直接作为实现与写作依据。

---

### 3. min/max 作用范围（归一化）

**原文定位：**
- 公式(5)：
  ```
  R = (E - min_{1≤j≤n} E_j) / (max_{1≤j≤n} E_j - min_{1≤j≤n} E_j)
  ```
- 说明："min_{1≤j≤n} E_j and max_{1≤j≤n} E_j are the minimum and maximum respectively of E when the number of complex networks is n."
- 第 3.2 节："R **standardizes the network structure entropy of different operating conditions into a unified framework**"

**复核情况：**
1. min/max 的取值范围是**全部 n 个复杂网络（即全部时间窗口）上的结构熵序列**，不是单个窗口内部。
2. 原文明确说把 "different operating conditions" 标准化到 "a unified framework"，说明是**跨工况统一归一化**，而非各工况独立归一化。

**结论：** 已确认。实现时应把所有工况（No Fault、Fault 4、Fault 11）的结构熵放在一起求全局 min/max，再统一归一化到 [0,1]。

**原文定位：**

- Lower

OCR 并未失真，只是语义上 "Lower" 比 "Medium" 更低，容易让人误解为"比 Low 还低"。实际按区间看，它是第二档（0.3–0.6），相当于 "Relatively Low"。

### 8. 注意力具体结构

**原文定位：**
- 第 2.2 节末尾，公式(4)：
  ```
  y_i = Σ_{j=1}^{T} a_{ij} h_j = Σ_{j=1}^{T} [exp(e_{ij}) / Σ_{k=1}^{T} exp(e_{ik})] h_j
  e_{ij} = a · <s_{i-1}, h_j>
  ```
- 说明："a_{ij} is the weight matrix containing pertinent information, and s is the output of the fully connected model layer."

**复核情况：**
这是典型的 **加性注意力（Additive Attention）**：
- 用前一步的全连接输出 s_{i-1} 与当前时间步隐藏状态 h_j 做点积（或内积）计算能量 e_{ij}
- 再经 softmax 得到权重 a_{ij}
- 最后对全部时间步隐藏状态 h_j 加权求和得到输出 y_i

不是点积注意力（Dot-Product / Scaled Dot-Product）。

**结论：** 已确认。可实现为“对全部时间步隐藏状态做加性 attention，再接全连接输出”。

### 10. 按工况分别训练还是合并训练

**原文定位：**
- 第 4.2.3 节："**9000 pieces of relative risk data under Fault 11 are chosen**, and the mean is 0.5418"
- 第 4.2.2 节：风险分布分析涉及 "No Fault, Fault 4, and Fault 11" 三工况

**复核情况：**

2. 预测训练阶段：论文明确说选了 **Fault 11 的 9000 条数据** 来做 Attention-Bi-LSTM 的拟合与训练。

**结论：** 已确认。预测模型是在 **单工况（Fault 11）** 上训练测试的；No Fault 和 Fault 4 主要用于风险分布对比与验证，不直接参与 Attention-Bi-LSTM 的训练集。首版可先按此口径执行：Fault 11 跑预测，三工况做风险序列与分布对比。
