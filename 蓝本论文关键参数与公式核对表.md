## 3. 小公式说明

### 3.1 结构熵

**复核后原文公式（Eq.1）：**

```text
E = - Σ_{i=1}^{N} (k_i / Σ_{i=1}^{N} k_i) · ln(k_i / Σ_{i=1}^{N} k_i)
```

其中：

- `k_i` 表示第 `i` 个节点的度；
- `N` 是网络节点总数；
- 每个滑动窗口会得到一张网络，也就会得到一个熵值；
- 这些熵值按时间顺序拼接后，形成后续的一维风险序列基础。

**当前结论（已确认）：**

- 公式排版、符号、定义均已人工复核 PDF 确认，与原文完全一致；
- 可直接作为实现与写作依据；
- 正式写论文时，建议把原文公式截图或手抄一遍，避免 OCR 公式污染最终稿。

### 3.2 相对风险序列

**复核后原文公式（Eq.5）：**

```text
R_t = (E_t - min_{1≤j≤n} E_j) / (max_{1≤j≤n} E_j - min_{1≤j≤n} E_j)
```

实现含义：

- `E_t` 是第 `t` 个窗口对应的结构熵；
- `min_{1≤j≤n} E_j` 和 `max_{1≤j≤n} E_j` 是全部 `n` 个复杂网络（全部时间窗口）上结构熵的最小值和最大值；
- 原文明确说 "standardizes the network structure entropy of **different operating conditions** into a unified framework"；
- `R_t` 是归一化后的相对风险值，范围收敛到 `[0,1]`，便于后续画图、分级和送入神经网络。

**当前结论（已确认，关键修正）：**

- **不是“单工况内独立归一化”，而是“跨工况统一归一化”**；
- 实现时把 No Fault、Fault 4、Fault 11 的结构熵拼在一起求全局 min/max，再统一计算 `R_t`。

### 3.3 风险分级

**复核后原文 Table 1：**

```text

0 < |R| ≤ 0.3         Low
0.3 < |R| ≤ 0.6       Lower

```

- `0.9 < |R| ≤ 1.0` -> `高风险`
